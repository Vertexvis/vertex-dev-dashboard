## Acceptance Criteria
- [x] A new Dev Dashboard route exists at /file-collections/:id.
- [ ] The File Collections table provides a clear way to view a file collection's files, either by making the collection name clickable, adding a View Files button or row action, or both.
- [ ] Selecting the clickable collection name, View Files button, or equivalent row action from the File Collections table navigates the user to /file-collections/:id for that file collection.
- [ ] The route supports direct navigation by URL using only the file collection ID.
- [ ] The file collection detail page includes simple return navigation, either a breadcrumb at the top of the page or a back button in the header, that takes the user back to the File Collections view.
- [ ] When a valid file collection ID is provided, the page loads and displays available metadata for the file collection.
- [ ] The page displays a files table containing the files associated with the file collection.
- [ ] The files table reuses the existing Files page client-side table behavior where practical.
- [ ] The files table includes the same columns and row interactions as the existing Files page unless specific fields or interactions are unavailable from the file collection files response.
- [ ] The files list is loaded from the available file collection files API: GET /file-collections/:id/files.
- [ ] If the file collection exists but contains no files, the page displays an empty files table.
- [ ] If the file collection ID is invalid or the file collection cannot be found, the page displays a 404 Not Found view.
- [ ] If the file collection metadata loads but the files request fails, the page still displays the collection metadata and renders an empty files list.
- [ ] If the files request fails, an exception is logged in the developer console.
